## Why "files + git + AI" works better than a dashboard

<!--
The core thesis:
- Project state lives in structured files (YAML frontmatter + markdown) in git repos
- Projects linked to strategic pillars, domains, contracts, vendors through references
- Version control gives you history, diffability, and accountability for free
- AI (Claude Code) can read, synthesize, and reason across the full project landscape
  in a way that no dashboard or Kanban board supports
- The tradeoffs are honest: you lose notifications, dashboards, and team collaboration features.
  You gain total ownership, AI-native access, and a system that works the way you think.

for an LLM with filesystem access.

## The three surfaces

<!--
How the work actually flows across different Claude interfaces:
- Claude Code: deep work sessions — analysis, writing, code, file management.

- Claude Mobile: capture — quick notes, voice memos, ideas that need to land

Each surface has a distinct role. The system works because they're complementary,

## What this doesn't solve

<!--
Honest limitations:
- Single-player system — this doesn't scale to a team without significant adaptation
- No real-time collaboration — git is async by nature
- Requires comfort with the command line and a mental model of how files and repos work
- The AI is a synthesizer, not a decision-maker — it surfaces patterns and connections,
  but judgment is still entirely human
- Bus factor of one: if the person maintaining this system leaves, the system leaves too
  (mitigated somewhat by everything being in version-controlled text files, but still real)

This section is important. The approach is genuinely useful but genuinely limited.
Don't undersell the limitations to make the concept sound more impressive.
